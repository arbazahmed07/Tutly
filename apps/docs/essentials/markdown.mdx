## Writing Style

When writing Tutly documentation, follow these guidelines:

### Headers

Use headers to organize content hierarchically:

```md
## Main Section

### Subsection

#### Minor Section
```

<Tip>
  Each header creates an anchor and appears in the table of contents on the
  right.
</Tip>

## Text Formatting

Use these formatting options to emphasize important information:

| Style         | How to write it   | Result          |
| ------------- | ----------------- | --------------- |
| Bold          | `**bold**`        | **bold**        |
| Italic        | `_italic_`        | _italic_        |
| Strikethrough | `~strikethrough~` | ~strikethrough~ |

## Links

- External links: `[link text](https://example.com)`
- Internal links: `[link text](/path/to/page)`

## Code Examples

Use code blocks to show API examples, configuration, or commands:

```bash
# Example command
tutly-cli login
```

## Notes and Warnings

Use these components to highlight important information:

<Note>This is a note with important information.</Note>

<Warning>This is a warning about potential issues.</Warning>

## Tables

Use tables to present structured information:

| Feature      | Description                 | Status      |
| ------------ | --------------------------- | ----------- |
| Live Classes | Built-in video conferencing | Available   |
| Assignments  | Test case-based evaluation  | Available   |
| Portfolios   | Student showcase pages      | Coming Soon |
